# 后台举报与内容治理前端设计

日期：2026-07-13
状态：已确认
范围：前端第一阶段，只实现举报工单管理与完整内容治理

## 1. 背景与目标

Discover World 后端已经提供举报工单、管理端内容查询、动态与评论隐藏恢复、论坛锁定和分区置顶能力。当前前端 `/admin` 只有首页配置、媒体审核和基于公开列表的基础内容治理，无法处理举报工单，也无法查询隐藏内容或治理评论。

本阶段目标：

- 在 `/admin` 内建立带侧边栏的管理工作区。
- 增加举报工单列表、详情和处理流程。
- 将内容治理切换到管理端内容列表，支持动态与评论治理。
- 保留现有论坛锁定、解锁和分区置顶能力。
- 保持现有中性、克制、内容优先的产品视觉语言。

本阶段不实现运营概览、标签运营、操作审计、账号管理和媒体审核增强。

## 2. 页面架构

后台继续使用单一路由 `/admin`，当前栏目通过查询参数保存：

```text
/admin?tab=homepage
/admin?tab=media-review
/admin?tab=reports
/admin?tab=moderation
```

未知或缺失的 `tab` 回退到 `homepage`。页签切换使用现有 SPA 导航模式，支持刷新、浏览器前进后退和复制链接。

### 2.1 后台壳

`AdminPage.tsx` 只负责：

- 管理员权限守卫。
- SidebarProvider 和后台整体布局。
- 从 URL 读取、校验和切换当前栏目。
- 渲染对应栏目面板。

侧边栏使用 `npx shadcn@latest add sidebar` 生成的官方 Sidebar 组件。桌面端保持常驻，移动端通过 SidebarTrigger 打开抽屉。

侧边栏栏目：

- 首页配置。
- 媒体审核。
- 举报工单。
- 内容治理。

侧边栏顶部显示“管理员控制台”，底部提供返回站点首页入口。不展示尚未实现的空栏目。

### 2.2 文件边界

计划拆分为：

- `frontend/src/pages/AdminPage.tsx`：权限守卫、后台壳、URL 栏目状态。
- `frontend/src/components/admin/AdminSidebar.tsx`：后台导航和移动端触发器。
- `frontend/src/components/admin/AdminReportsPanel.tsx`：举报列表、详情和处理。
- `frontend/src/components/admin/AdminContentModerationPanel.tsx`：动态与评论查询和治理。
- `frontend/src/components/admin/AdminForumModerationPanel.tsx`：论坛帖子治理。
- `frontend/src/lib/admin-navigation.ts`：栏目参数解析和 URL 构建纯函数。
- `frontend/src/lib/types.ts`：新增后台举报和内容治理类型。
- `frontend/src/lib/api.ts`：新增后台举报和内容治理请求函数。

现有首页配置和媒体审核代码本阶段只移动到内部渲染分支，不做无关重构。

## 3. 举报工单工作区

举报工单使用左右分栏布局。桌面端左侧列表、右侧详情；移动端改为上下布局，选择工单后将操作区滚动到可见位置。

### 3.1 列表与筛选

默认筛选 `status=open`，每页 20 条。支持：

- 状态：待处理、已采纳、已驳回、已处理、全部。
- 目标类型：动态、评论、全部。
- 举报人 ID。
- 目标 ID。
- 创建时间范围。

列表项显示举报原因、目标类型、举报人、状态和创建时间。首次加载自动选中当前页第一条；筛选或翻页后，如果当前选中项不在结果中，则选中新列表第一条。

### 3.2 详情

选中列表项后调用举报详情接口。详情区显示：

- 举报人账号摘要。
- 举报原因和补充说明。
- 目标类型和目标 ID。
- 当前状态。
- 处理人、处理结果、处理说明和处理时间。

已关闭工单只展示处理记录，不再显示提交按钮。

### 3.3 处理

待处理工单可选择：

- 采纳 `accepted`。
- 驳回 `rejected`。
- 已处理 `resolved`。

联动治理动作：

- 动态：不处理内容、隐藏、恢复、锁定论坛帖、解锁论坛帖。
- 评论：不处理内容、隐藏、恢复。

处理说明可选，但执行内容治理动作时前端要求填写说明，用于后台审计。提交时使用工单自身的目标类型和目标 ID，不允许用户手工改写。

成功后：

1. 使用接口返回值刷新详情。
2. 从默认待处理列表移除已关闭工单，或在其他状态筛选下更新对应列表项。
3. 使用 Sonner 显示处理结果。

失败后保留用户填写的结果、动作和说明，显示错误信息并允许重试。

## 4. 内容治理工作区

内容治理内部包含“动态与评论”和“论坛帖子”两个视图。

### 4.1 动态与评论

数据来源改为 `/api/admin/moderation/content/list`，不再使用公开动态接口代替后台查询。

支持：

- 类型：全部、动态、评论。
- 状态：全部、正常、隐藏。
- 作者 ID。
- 正文关键字。
- 服务端分页，每页 20 条。

左侧列表显示类型、作者、正文摘要、状态和创建时间；右侧显示完整内容、作者资料、目标 ID、状态和治理操作。

治理动作：

- 正常动态可隐藏，隐藏动态可恢复。
- 正常评论可隐藏，隐藏评论可恢复。

执行治理前必须填写原因。成功后更新当前行和详情状态，并使用 Sonner 提示；失败时保留原因。

### 4.2 论坛帖子

论坛帖子继续使用现有论坛列表接口，保留：

- 按论坛分区查看帖子。
- 锁定和解锁。
- 分区置顶和取消置顶。

论坛视图调整为和治理工作区一致的列表与操作区结构，但不改变后端语义。

## 5. API 与类型

前端新增以下接口封装：

```text
fetchAdminModerationReportList
fetchAdminModerationReportDetail
resolveAdminModerationReport
fetchAdminContentList
adminHideComment
adminRestoreComment
```

现有动态和论坛治理函数增加可选的 `reason` 与 `reportId` 字段，保持旧调用兼容。

新增请求和响应类型与 `api/discover_world.api` 一一对应：

- `AdminModerationReportQueryRequest`
- `AdminModerationReportDetailRequest`
- `AdminModerationReportResolveRequest`
- `AdminModerationReportResponse`
- `AdminModerationReportPageResponse`
- `AdminContentQueryRequest`
- `AdminContentResponse`
- `AdminContentPageResponse`

所有 ID 保持字符串类型，避免 JavaScript 大整数精度问题。

## 6. 视觉与响应式

使用场景是管理员在桌面端集中处理内容，默认采用明亮中性背景和较高信息密度。

视觉原则：

- 颜色策略为 restrained，中性色承担主体，强调色只用于当前导航、主操作和状态。
- 页面不使用渐变文字、装饰性玻璃效果或嵌套卡片。
- 列表与详情通过边界、留白和列宽区分，不为每行添加厚重卡片。
- 状态颜色仅表达业务语义：待处理、采纳、驳回、正常、隐藏。
- 所有图标按钮提供可访问名称，表单控件具备关联标签。

响应式：

- 大屏：Sidebar 常驻，工作区左右分栏。
- 中屏：Sidebar 可折叠，列表和详情保持分栏但调整宽度。
- 小屏：Sidebar 使用抽屉；工作区上下排列；筛选器允许换行。

不需要生成图片素材，后台工作区以数据和操作为主，现有组件即可表达完整层级。

## 7. 加载、空状态与错误处理

- 首次加载显示与实际列表结构匹配的骨架。
- 筛选请求期间保留现有结果并降低操作可用性，避免界面闪空。
- 空状态说明是“没有符合当前条件的内容”，并提供清除筛选入口。
- 列表失败显示页面内错误和重试按钮。
- 详情失败只影响右侧详情区，不清空左侧列表。
- 写操作期间禁用重复提交；关闭或切换栏目不会伪造成功状态。
- 401/403 沿用统一 API 错误信息，管理员权限守卫仍在页面入口生效。

## 8. 测试与验收

遵循测试先行：

1. 先增加 API 路径、请求参数和类型契约测试，确认测试因缺少实现而失败。
2. 增加后台栏目解析和 URL 构建的纯函数测试。
3. 增加 Sidebar 栏目、举报面板和内容治理面板的源码契约测试。
4. 实现最小代码使测试通过，再进行组件拆分和整理。

最终验证：

```bash
cd frontend
npm test
npm run lint
npm run build
```

浏览器验收：

- 桌面宽度验证 Sidebar 常驻、左右分栏、筛选、分页和处理反馈。
- 移动宽度验证 Sidebar 抽屉、上下布局和操作区可达性。
- 验证 `/admin?tab=...` 刷新、前进后退和未知参数回退。
- 验证举报处理、动态隐藏恢复、评论隐藏恢复和论坛操作成功后状态同步。

## 9. 明确不做

- 不实现运营概览、标签运营、操作审计和账号管理。
- 不增强相册、论坛分区或分享链接后端能力。
- 不修改普通用户举报入口。
- 不改动媒体上传、搜索、通知和个人主页。
- 不引入新的状态管理库或路由库。
- 不提前展示尚未实现的后台菜单项。
