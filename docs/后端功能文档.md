# 后端功能文档

> 更新时间：2026-07-13  
> 依据：当前仓库的 `api/discover_world.api`、`internal/handler`、`internal/logic`、`internal/svc`、`model`、`sql/create` 与 `etc/application.yaml`。  
> 本文只描述当前代码已经注册并实现的后端能力，不把前端功能、历史方案或仅有表结构的预留能力算作已完成接口。

## 1. 系统概览

Discover World 后端是一个基于 Go 与 go-zero 的 REST 单体服务，当前负责：

- 账号注册、登录、资料和头像管理。
- 用户关注关系、粉丝/关注列表和关注状态查询。
- 图片作品上传、查询、审核、互动、下载和删除。
- 用户动态、评论、点赞、收藏、置顶和可见范围控制。
- 论坛分区、论坛发帖、锁帖和分区置顶。
- 通知列表、未读数与已读状态管理。
- 关注用户的动态流和作品流。
- 用户主页动态、精选作品和相册列表。
- 全局搜索、首页 Hero/精选配置和站点概览统计。
- 内容举报、内容治理、标签运营、后台审计与权限控制。
- 动态和媒体上传时的 IP 属地快照。

当前 `api/discover_world.api` 共定义并注册 **73 个 POST 接口**：

| 接口范围 | 数量 | 访问要求 |
| --- | ---: | --- |
| 公开接口 | 10 | 无需登录 |
| 普通登录接口 | 38 | JWT |
| 管理端接口 | 25 | JWT + 管理员中间件，部分业务再校验 capability |

## 2. 技术架构

### 2.1 目录职责

```text
main.go                    服务入口
api/discover_world.api     go-zero API 定义
etc/application.yaml       主配置
etc/storage/default.yaml   默认对象存储密钥配置
internal/handler           HTTP 请求解析与响应输出
internal/logic             业务逻辑与事务编排
internal/middleware        管理员校验中间件
internal/common            JWT、响应、客户端 IP、IP 解析等公共能力
internal/svc               数据库连接、模型和公共依赖装配
internal/types             API 请求/响应结构
model                      MySQL 数据访问层
sql/create                 数据表结构
data/ipgeo                 ip2region IPv4/IPv6 离线库
```

### 2.2 请求处理链路

```text
HTTP 请求
  -> go-zero 路由/JWT 校验
  -> Client IP 中间件
  -> AdminCheck（管理接口）
  -> handler 解析参数
  -> logic 校验、鉴权、事务和业务编排
  -> model 访问 MySQL
  -> 统一 JSON 响应
```

### 2.3 服务启动与配置

`main.go` 默认读取 `etc/application.yaml`，可通过 `-f` 指定其他配置文件。启动时会：

1. 加载主配置。
2. 尝试加载同目录的 `.local.yaml` 覆盖 COS 配置。
3. 加载 `etc/storage/config/default.yaml` 或兼容路径 `etc/storage/default.yaml` 的存储密钥。
4. 初始化 MySQL 连接、全部业务 Model、管理员中间件和 IP 属地解析器。
5. 注册统一错误处理、CORS 和客户端 IP 中间件。
6. 注册 API 路由并启动 HTTP 服务。

主要配置项：

| 配置 | 用途 |
| --- | --- |
| `Name/Host/Port` | 服务名称和监听地址 |
| `MaxBytes/Timeout` | 请求体上限和请求超时 |
| `Mysql.DataSource` | MySQL 连接 |
| `CacheRedis` | Redis 配置，目前业务 ServiceContext 未显式接入缓存模型 |
| `Auth.AccessSecret/AccessExpire` | JWT 密钥和有效期 |
| `Admin` | 管理员账号配置预留 |
| `IpGeo` | IP 属地开关、可信代理、解析器和离线库路径 |
| `StorageSecrets` | 对象存储密钥映射 |

## 3. 通用后端能力

### 3.1 统一响应与错误码

成功响应格式：

```json
{
  "code": 200,
  "message": "成功",
  "data": {}
}
```

常用业务错误映射为 HTTP 400、401、403、404、409、500，并统一返回 `code` 与 `message`。

### 3.2 JWT 登录鉴权

- 登录成功后签发 HS256 JWT。
- token 包含 `userId`、`iat` 和 `exp`。
- `userId` 必须是正整数，0、小数、无效字符串均会被拒绝。
- 鉴权会查询 active 且未软删除的账号。
- 普通接口使用 go-zero JWT；需要手工解析时也支持 `Authorization: Bearer <token>`。

### 3.3 管理员与 capability 权限

管理端路由首先经过 `AdminCheckMiddleware`，要求当前账号 `role=admin`。部分后台 logic 还支持基于 `admin_role_policy` 的 capability 校验：

```text
admin.account.manage
admin.media.review
admin.content.moderate
admin.operation.manage
admin.audit.read
```

`role=admin` 默认拥有全部能力；其他角色可通过策略表扩展业务权限，但当前 `/api/admin` 路由仍先经过管理员中间件。

### 3.4 事务支持

`ServiceContext.Transact` 会在同一数据库事务中重新绑定所有 Model。当前以下业务使用事务保证数据一致性：

- 动态创建、更新、删除及图片关联。
- 论坛发帖及讨论扩展记录。
- 点赞、收藏、评论与统计更新。
- 直传会话初始化和完成。
- 媒体删除及对象状态更新。
- 个人主页和首页精选关系替换。

### 3.5 内容可见性

| 可见性 | 游客/未关注 | 已关注 | 作者本人 | 管理员 |
| --- | --- | --- | --- | --- |
| `public` | 可见 | 可见 | 可见 | 可见 |
| `followers` | 不可见 | 可见 | 可见 | 可见 |
| `private` | 不可见 | 不可见 | 可见 | 可见 |
| `unlisted` | 不可见 | 不可见 | 可见 | 可见 |

动态创建当前只接受 `public`、`followers`、`private`；媒体还支持 `unlisted`。

### 3.6 IP 属地快照

- 中间件默认使用 TCP 对端地址；只有来源属于 `TrustedProxies` 时才信任 `X-Forwarded-For` 或 `X-Real-IP`。
- 支持 `ip2region` IPv4/IPv6 离线库，也支持静态 CIDR 规则。
- 动态、论坛帖、普通媒体上传和直传完成时记录属地快照。
- 数据库存储国家、省、市、区县、运营商、展示文案和解析来源。
- 不保存明文 IP，只保存 HMAC-SHA256 后的 `ip_hash`。
- 动态、媒体详情和搜索结果可返回 `ipRegion`。

## 4. 账号与用户资料

### 4.1 注册与登录

- 校验邮箱格式、密码至少 8 位和两次密码一致。
- username 为空时使用邮箱前缀，校验邮箱和 username 唯一性。
- 使用 bcrypt 保存密码哈希，创建默认 `user_profile`。
- 新账号默认 `role=user`、`status=active`。
- 登录时拒绝 disabled、deleted 或无密码哈希的账号。
- 登录成功更新 `last_login_at`，返回 JWT、账号资料和头像 URL。

### 4.2 账号详情

- 不传目标 ID 时查询当前登录账号。
- 本人和管理员可查看完整资料及全部媒体审核统计。
- 查看其他 active 用户时隐藏 email、phone、待审核数和拒绝数。
- 返回作品数、公开作品数、粉丝数、关注数和当前用户是否已关注。
- 仅管理员可通过 email 查询账号。

### 4.3 账号更新与头像

- 普通用户只能更新自己；管理员可更新其他账号。
- 支持 username、email、password、nickname、bio。
- 管理接口还支持 role 和 status。
- 头像上传复用媒体存储能力，媒体用途为 `avatar`，成功后写入 `user_profile.avatar_asset_id`。
- 账号响应中的头像地址根据媒体对象和存储桶动态生成。

## 5. 关注关系与社交关系列表

- 关注、取消关注和关注状态查询。
- 禁止关注自己，只允许关注 active 用户。
- 新增关注成功时向被关注用户写入 `follow` 通知。
- 查询任意 active 用户的粉丝列表和关注列表。
- 列表使用游标分页，默认 20 条，最多 100 条。
- 列表只返回公开账号摘要，不返回 email。

## 6. 用户主页

### 6.1 主页动态

- 查询自己或指定用户的动态。
- 根据本人、管理员、已关注、普通访问者决定可见范围。
- 置顶动态优先，并使用包含置顶状态的游标分页。
- 返回动态图片、统计、点赞/收藏状态和 IP 属地。
- 默认 10 条，最多 50 条。

### 6.2 个人精选作品

- 每个用户最多配置 20 张精选作品。
- 只能选择本人已公开、审核通过、用途为 `work` 的图片。
- 精选顺序通过 `asset_link` 保存。
- 查询他人主页时也只返回仍然公开且审核通过的作品。

### 6.3 相册列表

- 查询自己或指定用户的相册。
- 本人和管理员可查看非公开相册，其他用户只看公开相册。
- 返回相册封面、图片数量、可见性和状态。
- 默认每页 12 条，最多 60 条。
- 当前仅提供相册列表查询，没有相册创建、编辑、删除或相册图片管理 API。

## 7. 媒体资源与图片作品

### 7.1 媒体用途和审核

媒体用途包括 `work`、`post`、`avatar`、`temp`。

- 普通用户上传 `work` 时初始审核状态为 `pending`。
- 管理员上传或非 `work` 用途默认 `approved`。
- 管理员可将审核状态更新为 `pending`、`approved`、`rejected`。
- 审核结果、审核人、审核时间写入 metadata，并向作者写入 `media_review` 通知。

### 7.2 三种上传方式

1. multipart 上传：图片流量经过应用服务，最大 30MB。
2. URL 上传：后端拉取 HTTP/HTTPS 图片，最大 10MB，超时 30 秒。
3. COS 直传：后端创建 10 分钟上传会话和签名，客户端直接 PUT 到 COS，完成后由后端 HEAD 校验对象并落库。

支持 `jpg`、`jpeg`、`png`、`webp`。上传过程会维护媒体资源、对象、直传会话、标签、统计、宽高、MIME、大小、主色、BlurHash、JPEG EXIF 和 IP 属地。

### 7.3 列表与筛选

公开媒体列表只返回 active、public、approved、`asset_usage=work` 且未删除的资源。

支持按 ID、标题、分类、标签、文件大小、宽高、宽高比、扩展名、作者、关键字和创建时间筛选。

- 普通分页默认 10 条，最多 300 条。
- 游标分页默认 30 条，最多 60 条。
- 游标列表支持 `latest`、`hot`、`rising`、`created`。
- `hot` 基于累计互动和时间衰减。
- `rising` 基于 `entity_stat_hourly` 的近期增量和前一时间窗变化。

### 7.4 URL、查看和互动

- 媒体 URL 根据存储桶 CDN/endpoint 和对象 key 动态生成。
- 支持原图、自动压缩、1200×1200 WebP 缩略图和指定尺寸居中裁剪。
- 媒体详情校验可见性并增加浏览数。
- 下载接口校验原图权限，返回 URL、文件名和大小，并增加下载数。
- 媒体点赞当前只支持 `like`，会更新总统计和小时统计，并向作者写入通知。

### 7.5 删除保护

- 只有资源所有者或管理员可以删除。
- 删除采用软删除，同时标记原始对象为 deleted。
- 被头像、相册封面、个人精选或其他阻断关系引用时拒绝删除。
- 仅被动态附件引用时要求 `force=true` 二次确认，删除后解除动态图片关联。

## 8. 动态、评论与互动

### 8.1 动态内容

- 动态正文最多 2000 个字符，正文和图片不能同时为空。
- 最多关联 9 张图片，location 最多 255 个字符。
- 动态类型为 `daily`、`travel_share`。
- 可见性为 `public`、`followers`、`private`。
- 创建和更新时校验图片属于作者、状态有效，并与动态可见性匹配。
- 支持创建、更新、软删除、详情、置顶和取消置顶。

### 8.2 公开动态流

- 无需登录，仅返回 active、public、未删除动态。
- 支持按正文关键字和动态类型过滤。
- 支持 `latest`、`hot`、`rising` 排序。
- 使用游标分页，默认 20 条，最多 50 条。
- 返回作者、图片、统计、最多 3 个点赞用户摘要和 IP 属地。

### 8.3 点赞、收藏、浏览和评论

- 动态点赞支持 `like`、`love`、`clap`、`wow`。
- 动态收藏为开关操作。
- 点赞、收藏和详情浏览会更新 `entity_stat` 与 `entity_stat_hourly`。
- 点赞和收藏成功时向动态作者写入通知，不给自己发通知。
- 评论最多 1000 个字符，评论前校验用户有权查看动态。
- 论坛帖锁定后禁止新增评论。
- 评论成功后增加评论统计、刷新论坛最后活跃时间，并向作者写入通知。
- 评论列表默认 20 条，最多 50 条。

## 9. 论坛

- 公开查询 active 论坛分区和论坛帖子。
- 论坛正文复用 `post`，标题、分区、锁定、分区置顶和最后活跃时间保存在 `post_discussion`。
- 论坛帖子固定为公开动态。
- 标题必填且最多 120 个字符；正文最多 2000 个字符；最多 9 张公开且已审核图片。
- 创建时在同一事务内写入 post、post_discussion、图片关系、统计和 IP 属地。
- 管理员可锁定/解锁、分区置顶/取消置顶。
- 当前没有论坛分区创建、修改或删除 API。

## 10. 通知

### 10.1 通知读取

- 查询当前用户通知列表和未读数量。
- 标记单条通知已读或全部已读。
- 通知列表按 ID 倒序游标分页，默认 20 条，最多 100 条。

### 10.2 已接入的通知事件

| `eventType` | 触发动作 | 目标类型 |
| --- | --- | --- |
| `follow` | 用户新增关注 | `user_account` |
| `post_comment` | 评论他人动态 | `post` |
| `post_reaction` | 点赞他人动态 | `post` |
| `post_favorite` | 收藏他人动态 | `post` |
| `media_reaction` | 点赞他人作品 | `media_asset` |
| `media_review` | 管理员审核作品 | `media_asset` |

通知当前由业务事务或业务逻辑直接写入 MySQL，没有消息队列、Outbox 或异步消费者。

## 11. 关注内容流

- 查询当前用户所关注账号发布的动态和作品。
- 动态流包含 `public` 和 `followers` 动态。
- 作品流包含 `public` 和 `followers`、审核通过、用途为 `work` 的作品。
- 返回当前用户的点赞/收藏状态。
- 默认 20 条，最多 60 条。
- 当前每次最多读取前 100 个关注用户作为内容源，超过 100 个关注对象时不会覆盖全部关注源。

## 12. 全局搜索

公开搜索接口支持 `media`、`post`、`album`、`user` 四类结果。

- 查询参数可使用 `q`，兼容 `searchText`。
- 可通过 `types` 指定搜索类型；默认搜索全部四类。
- 空关键字返回四类空数组。
- 每类默认最多 20 条，最大 50 条。
- 媒体只搜索公开、审核通过的正式作品。
- 动态只搜索公开 active 动态。
- 相册只搜索公开 active 相册。
- 用户只搜索 active、未删除账号。
- 搜索范围包括标题、正文、描述、文件名、位置、作者名、昵称和标签等字段。
- 结果综合精确匹配、前缀匹配和互动统计排序。
- 当前没有独立搜索游标，每类一次最多返回 `pageSize` 条。

## 13. 首页配置与站点统计

### 13.1 首页配置

- 公开读取首页 Hero 和精选作品。
- Hero 配置保存在 `site_config`，包含媒体 ID 和焦点坐标。
- Hero 媒体失效、非公开或未审核时，公开接口按未配置处理。
- 管理员可设置或清空 Hero，焦点坐标限制在 0～100。
- 首页精选通过 `asset_link(owner_type=site, link_role=homepage_featured)` 保存顺序。
- 最多 20 张，只允许公开且审核通过的正式作品。
- 管理端同时提供单个内容精选/取消精选和批量替换精选列表两种入口。

### 13.2 站点公开统计

公开统计接口返回：

```text
publicMediaAssetCount  公开审核通过作品数
creatorCount           active 用户数
publicPostCount        公开动态数
publicAlbumCount       公开相册数
```

## 14. 举报、内容治理与运营后台

### 14.1 用户举报与举报工单

- 登录用户可举报动态或评论，`targetType` 支持 `post`、`comment_record`。
- reason 必填且最多 80 个字符，description 最多 500 个字符。
- 新举报初始状态为 `open`。
- 管理员可按状态、目标、举报人和时间查询举报列表和详情。
- 处理结果支持 `accepted`、`rejected`、`resolved`。
- 处理举报时可同时执行隐藏/恢复动态、隐藏/恢复评论、锁定/解锁论坛帖。
- 保存处理人、处理结论、说明和处理时间。

### 14.2 内容治理

- 查询动态或评论的管理端内容列表。
- 隐藏/恢复动态。
- 隐藏/恢复评论。
- 锁定/解锁论坛帖。
- 论坛分区置顶/取消置顶。

### 14.3 标签运营

- 标签分页查询和筛选。
- 修改标签名称、slug、类型和状态。
- 合并标签：迁移 tagging 关系到目标标签并停用源标签。

### 14.4 运营概览

当前后台概览实际返回待审核媒体数、待处理举报数、active 用户数、公开媒体数和公开动态数。

响应结构中的 `recentMediaCount`、`recentPostCount`、`recentReportCount` 当前 logic 未计算，返回零值。

### 14.5 后台操作审计

- 查询后台操作日志列表和详情。
- 支持按操作人、action、目标类型、目标 ID 和时间范围筛选。
- 后台写操作可记录操作前快照、操作后快照、原因和元数据。
- 审计快照会移除键名包含 password、token、secret 的敏感字段。
- `RecordOperation` 支持记录客户端 IP，但多数调用未主动传入 `ClientIP`，因此该字段可能为空。

## 17. 当前能力边界

1. 后端是 go-zero REST 单体，没有拆分微服务。
2. 通知同步写 MySQL，没有 MQ、Outbox、重试消费者和实时推送通道。
3. Redis 已配置，但当前 ServiceContext 未初始化显式缓存层。
4. 相册只有列表接口，没有创建、修改、删除和相册内容管理接口。
5. 论坛分区只有查询接口，没有后台分区管理接口。
6. 全局搜索是 MySQL LIKE/关系查询，一次最多返回每类 50 条，没有搜索引擎、相关性索引或游标分页。
7. 关注流最多聚合前 100 个关注用户。
8. 后台运营概览的三个 recent 字段当前固定为零值。
9. `share_link` 和 `media_variant_rule` 仍属于数据层预留能力。
